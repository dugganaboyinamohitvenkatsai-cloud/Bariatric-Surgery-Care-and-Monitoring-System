## Project Overview
This project simulates a monitoring dashboard where patients/clinicians can:
* **Register and Login** with email, username, and password validation.
* Input **Pre-Surgery Baseline Data** (Weight, Height, BMI, medical conditions, surgeon approval).
* Perform weekly **Post-Surgery** check-ins (monitoring Weight, BP, and Blood Sugar) with health alerts.
* Analyze progress through **Interactive Charts** and Excess Weight Loss (EWL) percentages in the **Reports** section.
* Receive **Daily Motivational Quotes** via async API integration.

## Technical Stack
* **HTML5**: Semantic structure, forms with native validation (`required`, `min`, `max`, `minlength`), and semantic `<table>` elements.
* **Vanilla CSS3**: CSS Variables, Flexbox, CSS Grid, and Media Queries for mobile responsiveness.
* **JavaScript (ES6+)**: Local Storage for state management, Objects/Arrays, event listeners, async/await (`fetch`), and a custom toast notification system.
* **Chart.js**: Third-party library for dynamic weight progression chart.

## How it meets the Evaluation Rubric (COs)
1. **CO1**: Uses proper semantic HTML — headers, lists (sidebar navigation), linked stylesheets, and a polished visual design.
2. **CO2**: Implements advanced CSS (Grid/Flexbox/Media Queries). Uses `<form>` validation and semantic `<table>` rendering.
3. **CO3**: Uses JavaScript ES6+ features — arrow functions, Array methods (`forEach`, `reverse`, `slice`), and Objects.
4. **CO4**: Dynamic DOM manipulation (toast notification system, content injection). Uses `localStorage` for data persistence and `fetch()` for quotes API and avatar generation.
5. **CO5**: Developed in VS Code, hosted on GitHub with a clean structure and comprehensive README.
