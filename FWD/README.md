# Bariatric Surgery Monitoring and Tracking System

A student web-development project designed to digitally monitor and track a patient's health data before and after bariatric weight-loss surgery. It features dynamic data visualization and healthcare metrics designed explicitly for academic evaluation.

## 🚀 Live Demo
*Coming soon — deploy via GitHub Pages*

## 📖 Project Overview
This project simulates a monitoring dashboard where patients/clinicians can:
* Securely **Register and Login**.
* Input strict **Pre-Surgery Baseline Data** (Weight, Height, BMI).
* Perform weekly **Post-Surgery** check-ins (monitoring Weight, BP, and Blood Sugar drops).
* Analyze progress through **Interactive Charts** and Excess Weight Loss (EWL) percentages in the **Reports** section.

## 💻 Technical Stack
* **HTML5**: Uses structured semantic tags, forms with native validation (`required`, `min`, `max`), and structured data (`<table>`).
* **Vanilla CSS3**: Uses CSS Variables, Flexbox, CSS Grid layouts, and Media Queries for complete mobile responsiveness.
* **JavaScript (ES6+)**: Handles all logical flows. Includes Local Storage for state management, Objects/Arrays mapping, event listeners, and asynchronous fetching (`async/await`, `fetch`).
* **Chart.js**: Third-party library integration for dynamic graphing of recovery progress. 

## 🗺️ How it meets Evaluation Rubric (COs)
1. **CO1**: Uses proper, semantic HTML structures headers, lists (in sidebar), and well-organized linked stylesheets with a highly-polished aesthetic.
2. **CO2**: Implements advanced CSS (Grid/Flexbox/Media Queries). Incorporates extensive `<form>` validations, and utilizes semantic `<table>` rendering for logging.
3. **CO3**: Heavily utilizes JavaScript ES6+, arrow functions, Array structures, Iterators (`forEach`, `reverse`, `slice`), and Objects.
4. **CO4**: Manipulates DOM nodes dynamically. Synthesizes `localStorage` arrays and calls external API endpoints via asynchronous Javascript (`fetch()` for motivation quotes and avatars).
5. **CO5**: Developed locally in VS Code with a robust structure, clean architecture, GitHub ready, and includes a comprehensive README.md.
